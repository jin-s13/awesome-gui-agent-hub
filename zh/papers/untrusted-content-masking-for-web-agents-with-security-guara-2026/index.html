深度分析

创新点

  • 不可信内容屏蔽（UCM），在Web环境中恢复可信指令与不可信数据之间的信任边界
  • 利用文档对象模型（DOM）结构在不读取内容的情况下区分可信区域与不可信区域
  • 在代理处理页面之前屏蔽不可信区域，并通过具有严格权限分离的沙箱接口路由交互

方法

UCM利用DOM识别不可信区域，在代理观察页面之前将其屏蔽，并通过沙箱接口路由所有代理交互。这重新建立了可信指令与不可信内容之间的严格分离，使Web代理能够在感知和行动的同时与对抗性数据隔离。
