深度分析

创新点

  • 通过Context-as-Action（ConAct）实现的主动上下文管理，将上下文管理视为策略发出的第一类动作
  • 结构化上下文字段（折叠的动作历史、折叠的UI状态、最近步骤记录），在保持上下文紧凑的同时保留关键事实
  • 包含2,956条轨迹的MemGUI-3K数据集，每条轨迹标注了ConAct动作，用于监督训练和离线分析
  • 将ConAct集成到端到端长时域移动GUI智能体中的MemGUI-Agent架构

方法

MemGUI-Agent采用Context-as-Action（ConAct），策略同时发出UI动作和上下文管理动作，维护三个结构化上下文字段而非被动历史累积。构建了包含2,956条轨迹的MemGUI-3K数据集，并标注了ConAct动作，通过监督微调训练8B参数模型得到MemGUI-8B-SFT。在MemGUI-Bench和分布外MobileWorld基准上进行评估。

关键结果

MemGUI-8B-SFT在MemGUI-Bench上取得了开源数据下最佳的8B性能，并泛化到分布外的MobileWorld基准。
