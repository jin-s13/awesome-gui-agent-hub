深度分析

创新点

  • CUA-HandCrafted基准测试：793个轮次、24个多步骤网页任务、56个手工制作的攻击模板、8个攻击家族、4个系统提示配置
  • 可重复性审计显示，手工制作的提示注入模板对当前前沿CUA无效（0% ASR），通过提示消融将抵抗力追溯到模型权重
  • 跨领域比较揭示了领域条件安全性：浏览器加固的模型在编码代理中仍然容易受到技能注入攻击（高达100% ASR）
  • 论证文献中的高ASR源于RL优化的注入文本而非攻击类别，并且未发布的优化字符串使得已发表的数据不可重复

方法

作者构建了一个包含793个浏览器代理轮次的公开基准测试，结合了24个多步骤网页任务、56个手工制作的攻击模板（涵盖8个家族和4个系统提示配置）。他们评估了Claude Sonnet 4.6和GPT-5.4在多步骤攻击成功率上的表现，进行了提示消融以定位抵抗力，并在编码代理基准测试（SkillBench）上交叉引用结果以测试领域泛化能力。

关键结果

针对前沿CUA，手工制作的多步骤攻击实现了0/140的成功率（Clopper-Pearson 95%置信上限2.60%），抵抗力存在于模型权重中；然而，相同的模型在编码任务上对手工制作的技能注入攻击成功率高达100%，展示了领域条件安全性。

局限性

  • 仅测试了手工制作的攻击模板；未复现先前工作中的RL优化注入字符串，因此审计未直接衡量优化文本的影响。
  • 安全加固是领域条件的，不能泛化到浏览器表面之外，如编码领域的高脆弱性所示。
  • 基准测试仅限于24个网页任务和一个编码领域交叉参考，可能遗漏了其他CUA模态或攻击面。
