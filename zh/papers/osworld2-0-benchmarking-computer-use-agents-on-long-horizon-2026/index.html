深度分析

创新点

  • 引入OSWorld 2.0，一个包含108个长期真实世界计算机使用工作流的基准测试，人类完成中位时间约1.6小时。
  • 任务设计用于捕捉未被充分代表的真实世界现象：流式交互、动态环境、跨源推理、隐式状态推断和视觉空间精度。
  • 基于真实的输入工件和真实的有状态用户配置文件数据，并附带单独的安全报告以审计安全敏感执行。
  • 揭示了前沿智能体超越基本GUI控制的特定失败模式，例如丢失约束条件、错过任务中途信息、猜测和跳过验证。

方法

OSWorld 2.0包含108个端到端计算机使用任务，使用Claude Opus 4.7平均需要318次工具调用。智能体在500步的主要二元完成指标下进行评估，并报告部分得分。基准测试包括安全报告，并针对真实工作流中常见但在先前基准测试中未被充分代表的挑战现象。

关键结果

使用最大思考和批量工具调用的Claude Opus 4.8完成20.6%的任务（部分得分54.8%），而GPT-5.5停滞在13%左右。智能体在约束条件、任务中途信息、猜测、验证和隐藏状态恢复方面最为挣扎。
