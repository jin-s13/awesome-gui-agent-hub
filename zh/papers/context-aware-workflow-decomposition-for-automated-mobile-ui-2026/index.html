深度分析

创新点

  • 针对移动UI标注的上下文感知工作流分解，将任务划分为更小的阶段而非单步标注。
  • 使用结构化提示和模式约束的JSON输出来引导多模态大语言模型。
  • 针对不同UI元素类型定制的元素特定标注指令。
  • 系统评估工作流深度（一步、两步、四步、八步）和元素类别分组对标注质量的影响。

方法

本文提出了一种流水线，利用多模态大语言模型，通过结构化提示和模式约束的JSON输出，将移动UI标注分解为上下文感知的阶段。实验在专家标注的MUIAnno数据集上比较了一步、两步、四步和八步工作流，涵盖八种常见UI元素类型，测量精确率和召回率。额外的分组实验评估了元素类别分组对标注质量的影响。

关键结果

两步工作流实现了最高的精确率，而更深层次的分解提高了召回率但引入了更多误报。标注质量取决于工作流深度和元素类别的分组方式。

局限性

  • 更深层次的工作流分解增加了误报，表明存在精确率-召回率的权衡。
  • 结果基于单一数据集（MUIAnno）和八种特定UI元素类型，限制了泛化性。
  • 如摘要所述，输出仍然对提示设计和任务组织敏感。
  • 没有单一的工作流或分组策略是普遍最优的；性能随元素类别而变化。
