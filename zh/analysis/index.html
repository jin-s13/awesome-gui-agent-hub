研究路线

基准任务与数据集设计

提供多样化、真实的GUI交互任务（桌面、动态视频、拖拽）和环境，使不同模型和框架下的智能体能力具备可比性。

尚未覆盖：任务分布是否覆盖所有关键平台（移动端、跨设备）和真实用户工作流，以及任务难度分级是否匹配实际部署需求。
指标与压力测试设计

在任务成功率之外，通过观察效率、不确定性校准、细粒度子目标完成度和空间安全性来评估智能体表现。

尚未覆盖：这些代理指标与下游用户价值（如任务完成时间、错误恢复）之间的关联未经验证；不确定性量化方法的跨模型和跨任务鲁棒性有限。
可比基线与报告规范

通过统一评估协议、多框架模型比较和公开工件，使进展可审计，例如多智能体基准套件和系统性UQ研究。

尚未覆盖：基准结论在不同实现和算力预算下的稳定性，以及缺乏针对智能体评估的标准化可重复性报告规范。

研究共性

  • 所有研究均依赖多模态视觉‑语言模型作为核心智能体架构，并强调需要超越简单点击的丰富交互方式。
  • 普遍认为现有基准未能捕捉真实世界GUI使用的复杂性，特别是动态内容、多步工作流和观察控制。
  • 超越总体成功率的细粒度评估被广泛认可为必要，子目标完成度、观察成本和不确定性是反复出现的主题。

关键差异

  • 覆盖的交互类型：部分基准关注基于点击的定位，其他则关注拖拽、动态视频流或CLI‑GUI组合操作，反映了对用户界面基元的不同假设。
  • 评估环境保真度：真实macOS桌面（MacAgentBench） vs 基于浏览器的模拟（LivingScreen） vs 合成任务生成（ScaleCUA），导致有效性和可重复性之间的权衡不同。
  • 优化目标：几项工作提出用于纯评估的新基准，而另一些工作引入训练方法（弱监督、在线RL、技能库）并使用基准展示性能提升，形成评估者与构建者之间的分裂。
  • 先进指标的采用：不确定性量化和观察控制指标仅在一部分论文中被引入，尚未就是否应成为标准报告要求达成共识。

开放问题

  • 如何系统地将基准扩展到移动端、跨设备和多平台场景，同时保持可重复性和任务有效性？
  • 当前评估指标（子目标完成度、观察成本、UQ校准）如何映射到以用户为中心的结果，如任务完成时间、错误率和满意度？
  • 如何在当前存在过度观察或观察不足的动态界面智能体中量化并改进观察控制能力？
  • 哪些因素导致不确定性量化排名在跨模型和跨接口下迁移性弱，能否在不进行目标特定重新评估的情况下实现鲁棒的UQ？

研究路线

面向GUI代理的基准测试与评估框架（含不确定性量化）

在不同界面条件（桌面、动态视频、拖拽、不确定性）下衡量代理能力、覆盖范围和可靠性。

尚未覆盖：基准排名能否预测真实部署安全性；跨模型系列和接口的UQ排名迁移性较差；缺少超越单步定位的多步轨迹评估。
GUI代理的训练与缩放方法（弱监督、在线强化学习、技能库）

通过可扩展的训练范式提升数据效率和任务性能，减少对标注数据的依赖。

尚未覆盖：缩放规律能否正式预测失败模式；对分布外应用和长时域任务的泛化；对抗性用户交互的鲁棒性。
交互抽象与代理架构（主动上下文管理、多模态技能、动态屏幕代理）

捕捉复杂交互模态（拖拽、动态内容、多步工作流），并管理上下文以维持长时域性能。

尚未覆盖：交互原语的统一理论框架；架构选择在不同部署环境中是否最优；上下文管理策略能否防止关键信息丢失。

研究共性

  • 真实世界的GUI代理评估要求基准超越简单点击任务，覆盖动态界面、多模态反馈和细粒度指标。
  • 通过弱监督或可扩展训练实现数据效率是共同优先事项。
  • 不确定性量化正成为可靠代理部署的必要组成部分。

关键差异

  • 技能工件中视觉图形的必要性：VISUALSKILL显示多模态技能优于纯文本，但其他工作使用基于文本的技能库。
  • 训练范式：ScaleCUA使用在线强化学习，GUICrafter使用弱监督，MemGUI-Agent使用监督微调，对最优方法缺乏共识。
  • 评估范围：一些基准针对单步定位（Argus），另一些针对多步任务完成（MacAgentBench、LivingScreen），交互类型也存在差异（点击与拖拽，静态与动态）。

开放问题

  • 如何将基准性能正式关联到开放部署中的安全性和可靠性？
  • 缩放规律确保泛化到未见界面所需的必要条件是什么？
  • 不确定性量化能否在不损害任务成功率的情况下集成到代理决策循环中？
  • 覆盖所有GUI代理任务的最小且充分的交互原语集合是什么？

method

GUI agent models, planning, grounding, action prediction, training, or reasoning methods

63 方法质量 99 近期性 41 可复现性 50 主题相关性

文献综述归纳

研究路线

数据高效的GUI智能体训练

通过任务合成、在线RL和弱监督视觉定位，减少对昂贵人工标注的依赖，扩展智能体训练规模。

尚未覆盖：这些训练收益能否迁移到动态、长时域界面，以及奖励信号是否足够丰富以应对复杂任务。
上下文管理与高效观察

通过主动上下文压缩、关键帧选择和观察控制，处理长时域任务和动态内容，以提升智能体性能。

尚未覆盖：跨不同应用和平台的泛化能力，以及上下文紧凑性与任务关键信息保留之间的最佳平衡。
定位可靠性与交互多样性

将动作空间从点击扩展到拖拽、滚动等模态，并量化模型在定位中的置信度，以提升安全性和鲁棒性。

尚未覆盖：不确定性估计的跨模型迁移，以及将不确定性量化集成到多步智能体回路中以实现纠正动作。

研究共性

  • 当前GUI智能体普遍存在观察效率问题，要么消耗过多上下文，要么遗漏关键信息，限制了在动态和长时域任务上的表现。
  • 利用未标注数据或可验证的合成任务进行可扩展训练，是避免高昂标注成本并提升智能体能力的必要手段。
  • 评估环境与交互类型碎片化，需要覆盖多样化应用、动态内容和多模态动作的基准测试。

关键差异

  • 训练范式：部分工作采用任务合成的在线RL，其他工作则从静态截图进行弱监督课程学习，导致监督信号和样本效率的权衡不同。
  • 优化目标：一些方法聚焦于扩展动作空间（如拖拽交互），另一些则优化上下文管理（主动压缩）或关键帧提取，反映出对主要性能瓶颈的不同假设。
  • 评估场景：基准涵盖macOS桌面、短视频平台和移动设备，难以比较方法改进在不同领域的可迁移性。

开放问题

  • 如何训练智能体在内容动态变化的界面中主动平衡观察与动作？
  • 实现鲁棒定位所需的最少监督数据量是多少，不确定性量化能否可靠指示模型可能失败的时刻？
  • 当上下文管理等某一方面的改进部署到端到端智能体时，会如何与其他组件（如训练效率）相互作用？

system

Runnable GUI agent systems, frameworks, sandboxes, tool-use platforms, or orchestration layers

60 方法质量 99 近期性 40 可复现性 50 主题相关性

文献综述归纳

研究路线

视觉网页代理的在线多轮强化学习框架

在真实网站上以轨迹级成功信号训练GUI代理，减少对静态数据集的依赖，提升真实环境适应能力

尚未覆盖：在线探索中的安全风险、对多样化专业应用的可扩展性，以及探索与用户敏感状态暴露之间的权衡
从无标注截图的弱监督训练流水线

利用大规模无标注GUI数据进行视觉定位，再以少量高质量数据校准，大幅降低标注成本

尚未覆盖：自动提取的监督信号的质量控制、向未知应用领域的迁移，以及无标注数据偏差对代理行为的影响
分层多模态技能库与动作接地视觉记忆

通过保留视觉图形和存储动作相关图像裁剪，减少状态级与动作级错误，提升UI任务完成率

尚未覆盖：在持续演化的应用中自动构建与维护技能库，以及动作接地记忆能否泛化到非UI的具身任务
计算机使用代理的安全基准与领域条件审计

提供可复现的提示注入抵抗力评估，揭示安全加固是领域特异的而非通用的

尚未覆盖：安全能力在浏览器或编码之外的领域泛化、RL优化攻击的可复现性，以及用户敏感状态检测与代理接管协议的集成

研究共性

  • 当前GUI代理，即便是前沿模型，在长期真实世界任务上的成功率仍然很低，常因工作流阶段遗漏、错误传播和隐式状态误解而失败。
  • 引入结构化视觉记忆或多模态技能工件，相比朴素的全图上下文或纯文本技能，能够提升任务完成度。
  • 安全评估对部署至关重要，但现有安全加固不具备跨领域泛化能力：在浏览器上抵抗提示注入的代理，在编码任务上可能高度脆弱。
  • 在真实环境中的在线交互与训练为实现更好的泛化提供了路径，但带来了用户敏感数据与可靠奖励信号方面的风险。

关键差异

  • 视觉记忆设计：全图记忆可减少状态级失败，但会增加动作级错误；动作接地的图像裁剪则提升动作精度，但牺牲了更广的上下文。
  • 训练范式：一方主张在线多轮RL用轨迹级成功作为信号更真实，另一方则展示从海量无标注数据中弱监督定位，用极少量标准数据即可匹敌或超越RL方法。
  • 安全评估方法论：一方认为手工攻击模板不足，须用RL优化的攻击字符串进行有意义的审计；另一方则证明前沿模型已能抵抗手工攻击，指出已发表红队结果存在不可复现的缺口。
  • 基准范围：不同基准在任务时长（短浏览器任务 vs. 数小时专业工作流）和领域（通用Web vs. 专用专业软件）上存在差异，导致对代理能力和主要失败模式得出不同结论。

开放问题

  • 如何让安全加固跨浏览器、编码及其他GUI领域泛化，而不需要每领域重新训练？
  • 何种机制能够防止代理在数小时工作流中发生错误传播与目标漂移，且如何在没有人类介入的情况下进行评估？
  • 当前长期基准得出的结论，在多大程度上能迁移到基准未覆盖的未知专业软件和动态环境中？
  • 在线训练框架如何在探索真实环境时安全地避免暴露用户敏感状态，以及敏感状态检测应如何集成到代理循环中？

application

Domain applications of GUI agents, such as office automation, mobile apps, web tasks, software operation, or professional tools

67 方法质量 98 近期性 39 可复现性 55 主题相关性

文献综述归纳

研究路线

领域特定基准与评估体系

系统评估GUI代理在办公自动化、动态视频平台、拖拽交互和欺诈检测等真实任务上的能力，揭示性能差距与失败模式。

尚未覆盖：缺乏跨平台与跨交互模态的统一基准；现有基准多为静态快照，未与真实环境持续对齐；评估成本与可复现性仍存顾虑。
代理效率与可靠性增强方法

通过执行编译提升重复任务速度，借助历史感知视觉评判改善规划准确性，并利用保留视觉图形的多模态技能库提升任务成功率。

尚未覆盖：技能与编译程序的跨应用迁移；UI或工作流变化时的动态更新；评判模型对未见任务分布与边缘情况的鲁棒性。
超越点击范式的多模态交互扩展

将GUI代理扩展到拖拽操作、操作间内容变化的动态界面以及视频引导任务，弥补观察控制与交互基础能力的不足。

尚未覆盖：交互模态组合（如拖拽+动态界面）的泛化；人类水平的成本-准确性权衡；对真实世界流式传输与延迟约束的鲁棒性。
安全关键应用中的GUI代理

测试代理在无信誉捷径下的欺诈检测推理能力，揭示当前模型过度依赖可疑线索、在净化证据下难以保持良性召回的问题。

尚未覆盖：仅限于短信到网页欺诈的699个案例；未评估对抗鲁棒性、其他诈骗类型或实时部署；决策可解释性尚未涉及。

研究共性

  • 当前GUI代理，即使是先进的视觉语言模型，在应用于真实、领域特定任务时仍表现出显著性能差距，远非简化点击基准所能反映。
  • 视觉基础与历史感知推理对提升代理性能至关重要，但多数模型仍缺乏有效的观察控制与长程规划能力。
  • 捕捉多样化交互类型（如拖拽、动态内容、命令行+图形界面）并提供细粒度指标的基准，对于揭示单一指标掩盖的潜在弱点不可或缺。
  • 通过外部知识（技能、编译程序、评判模型）增强代理可带来持续收益，但此类增强方法跨平台、跨任务分布的泛化性尚未得到证实。

关键差异

  • 任务范围：部分工作聚焦静态桌面生产力（如macOS应用），其他关注动态娱乐界面（短视频应用），还有面向高风险安全任务（欺诈检测），导致评估优先级与成功标准各异。
  • 改进策略：方法分为预计算技能库（VISUALSKILL）、运行时执行轨迹编译（PreAct）以及在动作执行前干预的评判模型（HiViG），各自对任务重复性与代理架构的假设不同。
  • 效率与成功率：PreAct强调重复任务加速，而多数基准以任务成功率为优化目标，显示计算成本与准确率之间的权衡尚未被联合处理。
  • 评估真实性：FraudSMSWalker刻意抑制URL与信誉线索以测试纯粹推理，其他基准则保留全部真实信息，这反映了在应移除还是利用捷径线索上的不同立场。

开放问题

  • 单一代理架构能否在不做任务特定修改的情况下，处理点击、拖拽、动态内容、视频引导等全谱系交互模态？
  • 当底层应用UI或工作流发生变化时，技能与编译程序如何更新或迁移，而无需完整重训练或重编译？
  • 在动态界面中，观察频率与任务完成成本之间的最佳平衡点是什么？代理能否学会自适应控制自身观察率？
  • 现有基准能否充分预测真实世界部署的成功，尤其是欺诈检测等安全关键应用中，假阳性与假阴性的代价高度不对称？

Surveys, taxonomies, tutorials, position papers, or roadmap papers for GUI agents

59 方法质量 98 近期性 39 可复现性 47 主题相关性

文献综述归纳

研究路线

可验证任务合成下的在线RL训练

通过自动生成具有可验证完成条件的任务，扩展智能体训练数据，减少对人类演示的依赖。

尚未覆盖：合成任务是否覆盖真实世界多样性而不引入分布偏差，以及验证器本身是否足够鲁棒以充当奖励信号。
自主评估与噪声校正

用基于VLM的评估器替代人工判断，为RL微调提供可扩展的奖励信号，并建模评估器噪声以提高训练稳定性。

尚未覆盖：假设单张最终截图能捕获任务完成情况，在长程任务中可能失效；噪声校正模型依赖对假阳性/阴性率的经验估计，这些估计可能无法跨域迁移。
不确定性量化与安全性基准测试

通过事后不确定性方法衡量智能体动作的可靠性，并在受控条件下审计对抗提示注入的安全性。

尚未覆盖：不确定性排名在不同模型类别和接口间迁移性差；安全加固具有领域条件性，无法泛化，导致编码智能体在浏览器智能体被加固后仍然脆弱。
面向长程任务的主动上下文管理

在扩展的交互序列中维护紧凑、结构化的上下文，将上下文管理视为第一类动作，防止因历史溢出导致的性能下降。

尚未覆盖：该方法仅在移动GUI任务上验证；相同的上下文结构是否能泛化到具有不同UI模式的桌面或网页智能体尚未得到证明。

研究共性

  • 视觉-语言模型是感知和操作GUI屏幕的骨干，领域一致认为需要超越孤立的单步准确率的结构化评估。
  • 通过与环境的在线交互（无论是通过RL还是自主反馈）比在预收集演示上的静态监督微调更能有效提升智能体的鲁棒性。
  • 任务合成和自动化评估对于扩展训练至关重要，研究者普遍认同需要可验证的奖励或结果检查来避免奖励黑客行为。
  • 安全性和可靠性正在成为首要关注点，多篇论文汇聚于需要领域特定的安全评估和事后不确定性量化来检测不可靠动作。

关键差异

  • 部分工作关注单步基础准确率（如UI元素定位），而其他工作优化多步任务完成，导致不同的评估指标和失败模式。
  • 监督信号的选择各不相同：一些使用合成任务验证器作为二元奖励，另一些使用自主VLM评估器，还有少数依赖受约束的人类演示，每种方法对噪声和可扩展性的假设不同。
  • 安全评估在攻击面上存在差异：一项工作测试浏览器智能体上的手工提示注入，另一项则聚焦编码领域的技能注入，关于前沿智能体鲁棒性的结论相互矛盾。
  • 上下文管理策略从被动历史截断到主动结构化折叠不等，但对于智能体是自主决定保留什么，还是应强加固定模式，尚无共识。

开放问题

  • 在内部状态不可访问的闭源智能体上，能否适配在开源权重模型上有效的不确定性量化方法，所需的最小可观测接口是什么？
  • 当同一模型同时部署在多个接口（浏览器、移动端、桌面）时，GUI智能体的安全性如何退化，领域特定的加固能否统一？
  • 任务合成多样性与训练算法选择对最终智能体性能的相对贡献如何，我们如何设计可控实验来解耦这两者？
  • 在什么条件下，生成的教程或关键帧摘要能真正提升智能体学习效率，而不是简单提供一条捷径，从而损害对未见任务的泛化？
