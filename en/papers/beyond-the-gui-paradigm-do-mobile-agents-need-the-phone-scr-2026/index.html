Beyond the GUI Paradigm: Do Mobile Agents Need the Phone Screen?

arXiv 2026 57.2 method

TLDR

Argues mobile agents should use CLI over GUI, showing coding agents outperform GUI baselines on benchmarks and a new task suite.

Reasoning

Strengths include a novel paradigm shift, strong empirical results on multiple benchmarks, and an oracle ceiling analysis. Weaknesses are the limited scope to coding agents without mobile-specific training and potential lack of real-world deployment validation.

Deep Analysis

Innovations

  • Proposing CLI as a first-class paradigm for mobile agents alongside GUI
  • Demonstrating that off-the-shelf coding agents can serve as mobile CLI agents without mobile-specific post-training
  • Introducing the CLI-Advantage Task Suite covering tasks beyond GUI scope (bulk operations, multi-condition filtering, aggregation, cross-app workflows, hidden device state)
  • Providing oracle CLI solutions to establish performance ceilings and quantify room for improvement

Methodology

The paper evaluates three coding agents (Claude Code, Terminus-2, mini-swe-agent) across four model APIs on AndroidWorld and MobileWorld benchmarks, comparing them against three reproducible GUI baselines (GUI-Owl-1.5-32B, MAI-UI, Qwen3-VL-32B). It constructs oracle CLI solutions to determine performance ceilings and introduces the CLI-Advantage Task Suite with 45 templates across five categories to assess tasks beyond typical GUI capabilities.

Key Results

Claude Code (Opus 4.7) achieved 71.8% on AndroidWorld and 51.9% on MobileWorld, outperforming all GUI baselines (69.3/68.1/57.8% and 43.2/26.3/13.3% respectively). Oracle CLI solutions reached 88.8% and 86.3%, indicating substantial headroom. On the CLI-Advantage Task Suite, every CLI agent outperformed every GUI baseline across all five categories, using significantly fewer steps per task (10.7 vs. 18.6).

Limitations

  • Agents were not post-trained for mobile-specific tasks, potentially limiting their effectiveness on mobile nuances
  • Current CLI agents still fall significantly short of oracle performance, leaving substantial room for improvement
  • The CLI paradigm may not be suitable for tasks that inherently require visual or GUI-based interaction (though the CLI-Advantage suite targets some beyond-GUI intents)
