Deep Analysis

Innovations

  • Introduction of Ko-WideSearch, the first Korean breadth-search benchmark for exhaustive set enumeration by web agents.
  • Automated synthesize-and-verify pipeline for constructing breadth tasks with complete gold sets and attribute tables.
  • Difficulty tiers controlled by two independent structural knobs: table width and a 2-D composite key, causing cross-product membership to vary from 0% to 100%.
  • Normalization-aware comparator shared between gold construction and grading to prevent over-dropping of stable date and count columns due to formatting differences.

Methodology

The benchmark is built via an automated synthesize-and-verify pipeline that generates tasks requiring full membership and per-item attribute tables for set-parent entities (e.g., TV seasons, dynasties). It contains 228 tables over 190 entities across 16 categories, with three difficulty tiers set by independently varying table width and a 2-D composite key. Evaluation uses Item-F1, Column-F1, and Row-F1, and a normalization-aware comparator is shared between gold construction and grading to ensure stable columns are not unfairly penalized.

Key Results

Across 20 web agents, Item-F1 reached 92.8 but Row-F1 only 53.7, showing agents recover set members but fail to correctly fill attribute rows. Performance degrades steadily as difficulty knobs harden, and neither additional search steps nor higher spending closes the gap. Error analysis reveals that finding correct values for open-ended free-text cells is the main challenge, while cells with standard answers like dates or names are usually correct.

Limitations

  • The benchmark is limited to Korean, so findings may not generalize to other languages or multilingual settings.
  • The automated synthesize-and-verify pipeline may still produce imperfect gold sets despite the normalization-aware comparator, as certifying completeness and correctness remains inherently costly.
  • The difficulty knobs (table width and composite key) may not capture all real-world breadth challenges, such as dynamic or ambiguous set boundaries.
