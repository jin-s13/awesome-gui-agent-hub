Deep Analysis

Innovations

  • Context-aware workflow decomposition for mobile UI annotation, dividing the task into smaller stages instead of single-step annotation.
  • Use of structured prompts and schema-constrained JSON outputs to guide multimodal LLMs.
  • Element-specific annotation instructions tailored to different UI element types.
  • Systematic evaluation of workflow depth (one-step, two-step, four-step, eight-step) and element-class grouping effects on annotation quality.

Methodology

The paper proposes a pipeline that decomposes mobile UI annotation into context-aware stages using multimodal large language models with structured prompts and schema-constrained JSON outputs. Experiments compare one-step, two-step, four-step, and eight-step workflows on the expert-annotated MUIAnno dataset across eight common UI element types, measuring precision and recall. Additional grouping experiments assess the impact of element-class grouping on annotation quality.

Key Results

The two-step workflow achieves the highest precision, while deeper decomposition improves recall but introduces more false positives. Annotation quality depends on both workflow depth and how element classes are grouped.

Limitations

  • Deeper workflow decomposition increases false positives, indicating a precision–recall trade-off.
  • Results are based on a single dataset (MUIAnno) and eight specific UI element types, limiting generalizability.
  • Outputs remain sensitive to prompt design and task organization, as noted in the abstract.
  • No single workflow or grouping strategy is universally optimal; performance varies with element classes.
