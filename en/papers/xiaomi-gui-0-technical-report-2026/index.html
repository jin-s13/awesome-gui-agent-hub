Deep Analysis

Innovations

  • Real-device-dominant hybrid infrastructure that uses physical devices as primary execution environment and sandboxes as auxiliary support, aligning data collection, training, rollout, and evaluation distributions with real deployment.
  • Multi-source training data covering high-frequency head tasks, high-generalization data for long-tail intents, and capability-enhancement data for reflection and memory.
  • Error-driven data flywheel that converts failure trajectories into corrected actions, reflective explanations, and recovery demonstrations.
  • Progressive three-stage training pipeline: supervised fine-tuning, step-level reinforcement learning, and agentic reinforcement learning.

Methodology

Xiaomi-GUI-0 is a native multimodal GUI agent trained and evaluated in a real-device closed loop. It uses a real-device-dominant hybrid infrastructure where physical devices are the primary environment, with sandboxes as auxiliary support. Training data is constructed from multiple sources and augmented via an error-driven flywheel, and the model is trained through a progressive pipeline of supervised fine-tuning, step-level RL, and agentic RL.

Key Results

Xiaomi-GUI-0 achieves 72.0% success on the in-house RealMobile benchmark and 78.9% on AndroidWorld, with substantial improvements in execution stability and abnormal-state recognition in real-world tasks.
