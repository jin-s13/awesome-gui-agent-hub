Deep Analysis

Innovations

  • ChainWorld framework for composing atomic OSWorld tasks into long-horizon desktop workloads via directional compatibility search
  • Introduction of single-turn and multi-turn evaluation protocols for long-horizon tasks
  • Identification of distinct failure profiles: artifact precision in single-turn vs. session management issues in multi-turn

Methodology

ChainWorld composes atomic OSWorld tasks into chains of length 2-4 using directional compatibility search while preserving source evaluators, yielding 347 chains. Four computer use agents are evaluated under two protocols: single-turn (all tasks in one prompt) and multi-turn (tasks revealed sequentially).

Key Results

Maximum chain completion across four agents is 31%; multi-turn evaluation improves completion for three models but both protocols remain challenging. Single-turn failures concentrate on artifact precision, while multi-turn failures reflect session management problems like fragmented progress and later turn disengagement.

Limitations

  • Both single-turn and multi-turn protocols remain challenging for current agents
  • Maximum chain completion is only 31% across evaluated agents
