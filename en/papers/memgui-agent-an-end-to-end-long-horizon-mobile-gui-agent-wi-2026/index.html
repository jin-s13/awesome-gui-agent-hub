MemGUI-Agent: An End-to-End Long-Horizon Mobile GUI Agent with Proactive Context Management

arXiv 2026 70.4 method, system

TLDR

MemGUI-Agent uses proactive context management (ConAct) to improve long-horizon mobile GUI agent performance, achieving state-of-the-art results on MemGUI-Bench and MobileWorld.

Reasoning

The paper introduces a novel context management approach (ConAct) that treats context as actions, addressing prompt explosion in long-horizon tasks. Strengths include a new dataset (MemGUI-3K) and strong empirical results on multiple benchmarks. Weaknesses are the focus on mobile agents only and reliance on a single 8B model scale.

Deep Analysis

Innovations

  • Proactive context management via Context-as-Action (ConAct) that treats context management as first-class actions emitted by the policy
  • Structured context fields (folded action history, folded UI state, recent step record) to preserve critical facts while keeping context compact
  • MemGUI-3K dataset with 2,956 trajectories annotated with ConAct actions for supervised training and offline analysis
  • MemGUI-Agent architecture integrating ConAct into an end-to-end long-horizon mobile GUI agent

Methodology

MemGUI-Agent uses Context-as-Action (ConAct) where the policy emits both UI actions and context management actions, maintaining three structured context fields instead of passive history accumulation. A dataset MemGUI-3K (2,956 trajectories) with ConAct annotations is constructed, and an 8B model is trained via supervised fine-tuning to produce MemGUI-8B-SFT. Evaluation is on MemGUI-Bench and the out-of-distribution MobileWorld benchmark.

Key Results

MemGUI-8B-SFT achieves the best open-data 8B performance on MemGUI-Bench and generalizes to the out-of-distribution MobileWorld benchmark.
