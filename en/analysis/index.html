Literature review synthesis

Research Lines

Benchmark task and dataset design

Providing diverse, realistic GUI interaction tasks (desktop, dynamic video, drag) and environments that make agent capabilities comparable across models and frameworks.

Open edge: Whether the task distribution covers all critical platforms (mobile, cross‑device) and real‑world user workflows, and whether task difficulty scales match actual deployment demands.
Metric and stress‑test design

Evaluating agent performance beyond task success rate by measuring observation efficiency, uncertainty calibration, fine‑grained sub‑goal completion, and spatial safety.

Open edge: The connection between these proxy metrics and downstream user value (e.g., task completion time, error recovery) is unverified; cross‑model and cross‑task robustness of uncertainty quantification methods is limited.
Comparable baselines and reporting

Enabling auditable progress through unified evaluation protocols, multi‑framework model comparisons, and publicly released artifacts, as seen in multi‑agent benchmark suites and systematic UQ studies.

Open edge: Stability of benchmark conclusions across different implementations and compute budgets, and the absence of standardized reproducibility reporting for agent evaluations.

Shared Direction

  • All studies rely on multimodal vision‑language models as the core agent architecture and highlight the need for richer interaction modalities beyond simple point‑and‑click.
  • There is agreement that current benchmarks fail to capture the complexity of real‑world GUI use, especially dynamic content, multi‑step workflows, and observation control.
  • Fine‑grained evaluation beyond aggregate success rate is widely recognized as essential, with sub‑goal completion, observation cost, and uncertainty being recurring themes.

Key Differences

  • Interaction types covered: some benchmarks focus on click-based grounding, others on drag, dynamic video feeds, or combined CLI‑GUI operations, reflecting different assumptions about user interface primitives.
  • Evaluation environment fidelity: real macOS desktop (MacAgentBench) versus browser‑based simulation (LivingScreen) versus synthetic task generation (ScaleCUA), leading to different validity and reproducibility trade‑offs.
  • Optimization target: several works propose new benchmarks for pure evaluation, while others introduce training methods (weakly supervised, online RL, skill libraries) that use benchmarks to demonstrate performance gains, creating a split between evaluators and builders.
  • Uptake of advanced metrics: uncertainty quantification and observation‑control metrics are introduced in only a subset of papers, and there is no consensus on whether these should become standard reporting requirements.

Open Questions

  • How can benchmarks be systematically extended to mobile, cross‑device, and multi‑platform scenarios while maintaining reproducibility and task validity?
  • What is the mapping from current evaluation metrics (sub‑goal completion, observation cost, UQ calibration) to user‑centric outcomes such as task completion time, error rate, and satisfaction?
  • How can observation control capability be quantified and improved in agents that currently over‑observe or under‑observe dynamic interfaces?
  • What factors cause the observed weak cross‑model and cross‑interface transferability of uncertainty quantification rankings, and can robust UQ be achieved without target‑specific re‑evaluation?

Research Lines

Benchmarking and evaluation frameworks for GUI agents (including uncertainty quantification)

Measures agent capabilities, coverage, and reliability under varying interface conditions (desktop, dynamic video, drag-based, uncertainty).

Open edge: Whether benchmark rankings predict real-world deployment safety; transfer of UQ rankings across model families and interfaces is weak; no multi-step trajectory evaluation beyond single-step grounding.
Training and scaling methods for GUI agents (weak supervision, online RL, skill libraries)

Improves data efficiency and task performance through scalable training paradigms, reducing reliance on annotated data.

Open edge: Formal guarantees that scaling laws predict failure modes; generalization to out-of-distribution applications and long-horizon tasks; robustness to adversarial user interactions.
Interaction abstraction and agent architecture (proactive context management, multimodal skills, living-screen agents)

Captures complex interaction modalities (drag, dynamic content, multi-step workflows) and manages context to sustain long-horizon performance.

Open edge: Unified theoretical framework for interaction primitives; whether architecture choices are optimal across deployment environments; verification that context management prevents critical information loss.

Shared Direction

  • Real-world GUI agent evaluation requires benchmarks beyond simple click-based tasks, covering dynamic interfaces, multimodal feedback, and fine-grained metrics.
  • Data efficiency through weakly supervised or scalable training is a shared priority.
  • Uncertainty quantification is emerging as a necessary component for reliable agent deployment.

Key Differences

  • The necessity of visual figures in skill artifacts: VISUALSKILL shows multimodal skills outperform text-only, but other works use text-based skill libraries.
  • Training paradigm: ScaleCUA uses online RL, GUICrafter uses weak supervision, MemGUI-Agent uses SFT; no consensus on optimal approach.
  • Evaluation scope: Some benchmarks target single-step grounding (Argus), others multi-step task completion (MacAgentBench, LivingScreen), and interaction types vary (click vs. drag, static vs. dynamic).

Open Questions

  • How can we formally connect benchmark performance to safety and reliability in open-ended deployments?
  • What are the necessary conditions for scaling laws to guarantee transfer across unseen interfaces?
  • Can uncertainty quantification be integrated into agent decision loops without compromising task success?
  • What is a minimal and sufficient set of interaction primitives that covers all GUI agent tasks?

Literature review synthesis

Research Lines

Data-efficient training for GUI agents

Scaling agent training by reducing reliance on costly human annotations through task synthesis, online RL, and weakly-supervised visual grounding.

Open edge: Whether these training gains transfer to dynamic, long-horizon interfaces and whether the reward signals are sufficiently rich for complex tasks.
Context management and efficient observation

Handling long-horizon tasks and dynamic content by proactive context compression, keyframe selection, and observation control to improve agent performance.

Open edge: Generalizability across diverse applications and platforms, and the optimal balance between context compactness and task-critical information retention.
Grounding reliability and interaction diversity

Expanding beyond click-based actions to drag, scroll, and other modalities, and quantifying model confidence in grounding to improve safety and robustness.

Open edge: Cross-model transfer of uncertainty estimates, and integration of uncertainty quantification into multi-step agent loops for corrective actions.

Shared Direction

  • Current GUI agents suffer from observation inefficiency, either consuming excessive context or missing critical information, which limits performance on dynamic and long-horizon tasks.
  • Scalable training methods that leverage unannotated data or verifiable synthetic tasks are essential to improve agent capabilities without prohibitive annotation costs.
  • Evaluation remains fragmented across different environments and interaction types, necessitating benchmarks that cover diverse apps, dynamic content, and multi-modal actions.

Key Differences

  • Training paradigm: Some works adopt online RL with task synthesis, while others use weakly-supervised curriculum learning from static screenshots, leading to different supervision signals and sample efficiency trade-offs.
  • Optimization target: Some methods focus on expanding the action space (drag interactions), while others optimize context management (proactive compression) or keyframe extraction, reflecting different assumptions about the primary performance bottleneck.
  • Evaluation setting: Benchmarks range from macOS desktops to short-video platforms and mobile devices, making it difficult to compare the transferability of method improvements across domains.

Open Questions

  • How can agents be trained to balance observation and action in highly dynamic interfaces where content changes between actions?
  • What is the minimal amount of supervised data required to achieve robust grounding, and can uncertainty quantification reliably indicate when a model is likely to fail?
  • How do improvements in one aspect (e.g., context management) interact with other components (e.g., training efficiency) when deployed in end-to-end agents?

Literature review synthesis

Research Lines

Online multi-turn reinforcement learning frameworks for visual web agents

enables training GUI agents on live websites with trajectory-level success signals, reducing the need for static datasets and improving real-world adaptability

Open edge: safety risks during online exploration, scalability to diverse professional applications, and the trade-off between exploration and user-sensitive state exposure
Weakly-supervised training pipelines from unannotated screenshots

drastically reduces annotation cost by leveraging large-scale unlabeled GUI data for visual grounding, then calibrating with minimal high-quality data

Open edge: quality control of automatically extracted supervision signals, transfer to unseen application domains, and the effect of unlabeled data biases on agent behavior
Hierarchical multimodal skill libraries with action-grounded visual memory

improves agent performance on UI tasks by retaining visual figures and storing action-relevant image crops, mitigating state-level and action-level errors

Open edge: automatic construction and maintenance of skill libraries across evolving applications, and whether action-grounded memory generalizes to non-UI embodied tasks
Safety benchmarking and domain-conditioned auditing of computer-using agents

provides reproducible evaluation of prompt injection resistance and reveals that safety hardening is domain-specific, not universal

Open edge: generalization of safety beyond browser or coding domains, reproducibility of RL-optimized attacks, and integration of user-sensitive state detection into agent handover protocols

Shared Direction

  • Current GUI agents, even frontier models, achieve low success rates on long-horizon real-world tasks, often failing due to workflow stage omission, error propagation, and hidden state misunderstanding.
  • Incorporating structured visual memory or multimodal skill artifacts improves task completion over naive full-image context or text-only skills.
  • Safety evaluation is essential for deployment, but existing safety hardening does not transfer across domains; agents resistant to browser prompt injection can be highly vulnerable on coding tasks.
  • Online interaction and training on live environments provide a path to better generalization, but introduce risks around user-sensitive data and reliable reward signals.

Key Differences

  • Visual memory design: full-image memory can reduce state-level failures but increases action-level errors, while action-grounded crops improve action precision at the cost of broader context.
  • Training paradigm: some works advocate online multi-turn RL with trajectory-level success as more realistic, while others show that weakly-supervised grounding from massive unlabeled data can match or outperform RL-based methods with far fewer annotations.
  • Safety evaluation methodology: one line of work argues that hand-crafted injection templates are insufficient and that RL-optimized attack strings are needed for meaningful auditing; another demonstrates that frontier models are already robust to hand-crafted attacks, highlighting a reproducibility gap in published red-teaming results.
  • Benchmark scope: benchmarks differ in task horizon (short browser tasks vs. multi-hour professional workflows) and domain (generic web vs. specialized professional software), leading to different conclusions about agent capabilities and primary failure modes.

Open Questions

  • How can safety hardening be made to generalize across browser, coding, and other GUI domains without requiring per-domain retraining?
  • What mechanisms can prevent error propagation and objective drift in agents executing workflows lasting hours, and how can these be evaluated without human-in-the-loop?
  • To what extent do the conclusions from current long-horizon benchmarks transfer to unseen professional software and dynamic environments not covered by the benchmarks?
  • How can online training frameworks safely explore live environments without exposing user-sensitive states, and how should sensitive-state detection be integrated into the agent loop?

application

Domain applications of GUI agents, such as office automation, mobile apps, web tasks, software operation, or professional tools

67 Methodology quality 98 Recency 39 Reproducibility 55 Topical relevance

Literature review synthesis

Research Lines

Domain-specific benchmark suites and evaluation frameworks

Systematically assesses GUI agent capabilities on real-world tasks from office automation, dynamic video platforms, drag interactions, and fraud detection, revealing performance gaps and failure modes.

Open edge: No unified benchmark across platforms and interaction modalities; benchmarks are mostly static snapshots lacking continuous alignment with evolving real-world environments; evaluation cost and reproducibility remain concerns.
Agent efficiency and reliability enhancement methods

Improves agent speed on repeated tasks via execution compilation, planning accuracy via history-aware visual critics, and task success via multimodal skill libraries that retain visual figures.

Open edge: Cross-application transfer of skills and compiled programs; dynamic updating when UIs or workflows change; robustness of critics to unseen task distributions and edge cases.
Multimodal interaction expansion beyond click-based paradigms

Extends GUI agents to drag-based operations, dynamic content that changes between actions, and video-guided tasks, addressing missing observation control and interaction grounding.

Open edge: Generalization across combinations of interaction modalities (e.g., drag + dynamic screen); human-level cost-accuracy trade-offs; robustness to real-world streaming and latency constraints.
Safety-critical application of GUI agents

Tests agentic reasoning for fraud detection without reputation shortcuts, revealing that current models over-rely on suspicious cues and struggle with benign recall under sanitized evidence.

Open edge: Limited to SMS-to-webpage fraud with 699 cases; no evaluation of adversarial robustness, other scam types, or real-time deployment; explainability of agent decisions remains unaddressed.

Shared Direction

  • Current GUI agents, even advanced VLMs, exhibit significant performance gaps when applied to real-world, domain-specific tasks beyond simplified click-based benchmarks.
  • Visual grounding and history-aware reasoning are critical for improving agent performance, yet most models lack effective observation control and long-horizon planning.
  • Benchmarks that capture diverse interaction types (e.g., drag, dynamic content, CLI+GUI) and fine-grained metrics are necessary to reveal hidden weaknesses that single-pass metrics obscure.
  • Enhancing agents with external knowledge (skills, compiled programs, critics) yields consistent gains, but the generalizability of these enhancements across platforms and task distributions is unproven.

Key Differences

  • Task scope: some works focus on static desktop productivity (e.g., macOS apps), others on dynamic, entertainment-oriented interfaces (short-video apps), and still others on high-stakes safety tasks (fraud detection), leading to different evaluation priorities and success criteria.
  • Improvement strategy: methods diverge into pre-computed skill libraries (VISUALSKILL), runtime compilation of execution traces (PreAct), and critic models that intervene before action execution (HiViG), each with different assumptions about task repetition and agent architecture.
  • Efficiency vs. success: PreAct emphasizes speedup on repeated tasks, while most benchmarks optimize for task success rate, indicating a trade-off between computational cost and accuracy that is not jointly addressed.
  • Evaluation realism: FraudSMSWalker deliberately suppresses URL and reputation cues to test pure reasoning, whereas other benchmarks retain all real-world information, reflecting different positions on whether shortcuts should be removed or leveraged.

Open Questions

  • Can a single agent architecture handle the full spectrum of interaction modalities (click, drag, dynamic content, video guidance) without task-specific modifications?
  • How can skills and compiled programs be updated or transferred when the underlying application UI or workflow changes, without requiring full re-training or re-compilation?
  • What is the right balance between observation frequency and task completion cost in dynamic interfaces, and can agents learn to adaptively control their own observation rate?
  • Are current benchmarks sufficient to predict real-world deployment success, especially for safety-critical applications like fraud detection where the cost of false positives and false negatives is highly asymmetric?

Surveys, taxonomies, tutorials, position papers, or roadmap papers for GUI agents

Literature review synthesis

Research Lines

Online RL training with verifiable task synthesis

Scaling agent training data via automatically generated tasks with verifiable completion criteria, reducing dependence on human demonstrations.

Open edge: Whether the synthesized tasks cover real-world diversity without introducing distributional bias, and whether the verifier itself is robust enough to serve as a reward signal.
Autonomous evaluation and noise correction

Replacing human judgment with VLM-based evaluators to provide scalable reward signals for RL fine-tuning, and modeling evaluator noise to improve training stability.

Open edge: The assumption that a single final screenshot captures task completion may fail for long-horizon tasks; the noise correction model relies on empirical estimates of false positive/negative rates that may not transfer across domains.
Uncertainty quantification and safety benchmarking

Measuring the reliability of agent actions through post-hoc uncertainty methods, and auditing safety against prompt injection attacks under controlled conditions.

Open edge: Uncertainty rankings transfer poorly across model classes and interfaces; safety hardening is domain-conditioned and does not generalize, leaving coding agents vulnerable even when browser agents are hardened.
Proactive context management for long-horizon tasks

Maintaining compact, structured context over extended interaction sequences by treating context management as a first-class action, preventing performance degradation from history overflow.

Open edge: The approach has been validated on mobile GUI tasks; whether the same context structure generalizes to desktop or web agents with different UI patterns is not yet shown.

Shared Direction

  • Vision-language models serve as the backbone for perceiving and acting on GUI screens, and the field agrees on the need for structured evaluation beyond isolated single-step accuracy.
  • Online interaction with environments, whether through RL or autonomous feedback, is more effective than static supervised fine-tuning on pre-collected demonstrations for improving agent robustness.
  • Task synthesis and automated evaluation are critical for scaling training, with shared agreement that verifiable rewards or outcome checks are necessary to avoid reward hacking.
  • Safety and reliability are emerging as first-class concerns, with papers converging on the need for domain-specific safety evaluation and post-hoc uncertainty quantification to detect unreliable actions.

Key Differences

  • Some works focus on single-step grounding accuracy (e.g., UI element localization) while others optimize multi-step task completion, leading to different evaluation metrics and failure modes.
  • The choice of supervisory signal varies: some use synthetic task verifiers as binary rewards, others use autonomous VLM evaluators, and a few rely on constrained human demonstrations, each with different assumptions about noise and scalability.
  • Safety evaluations differ in attack surface: one work tests hand-crafted prompt injection on browser agents, while another focuses on coding-domain skill injection, revealing conflicting conclusions about the robustness of frontier agents.
  • Context management strategies range from passive history truncation to proactive structured folding, but there is no agreement on whether the agent should decide what to keep or whether a fixed schema should be imposed.

Open Questions

  • Can uncertainty quantification methods that work for open-weight models be adapted to closed-source agents where internal states are inaccessible, and what is the minimal observable interface needed?
  • How does the safety of a GUI agent degrade when the same model is deployed across multiple interfaces (browser, mobile, desktop) simultaneously, and can domain-specific hardening be unified?
  • What is the relative contribution of task synthesis diversity versus training algorithm choice in the final agent performance, and how can we design controlled experiments to disentangle them?
  • Under what conditions do generated tutorials or keyframe summaries actually improve agent learning efficiency, as opposed to simply providing a shortcut that harms generalization to unseen tasks?
